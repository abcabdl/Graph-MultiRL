# AGENTS.md — GraphCredit-Offline on Dr. MAS

> Project: GraphCredit-Offline  
> Base repo: Dr. MAS  
> Goal: Build a structure-agnostic, offline, verifiable node-level credit assignment layer on top of Dr. MAS.  
> Target environment: offline server, no external internet, no external LLM judge, no commercial API.

---

## 0. Core mission

Build GraphCredit-Offline by modifying Dr. MAS.

GraphCredit-Offline should extend Dr. MAS from:

```text
multi-agent RL with agent-wise advantage normalization
```

to:

```text
multi-agent RL with event-graph-based node-level credit assignment
+ offline verifiable process reward
+ counterfactual marginal credit
+ agent-node-type advantage normalization
```

The system must not depend on an external LLM coach.

The core research claim is:

```text
Dr. MAS stabilizes multi-agent RL by normalizing advantage per agent,
but its credit assignment is still coarse: each agent receives trajectory-level or agent-level reward signals.

GraphCredit-Offline extends Dr. MAS by converting multi-agent rollouts into event graphs,
assigning credit to action/message/tool-call/router/verifier/final-answer nodes,
and stabilizing training with agent-node-type advantage normalization.
```

---

## 1. Important constraints

The server cannot access the internet.

Therefore:

```text
1. Do not call GPT, Claude, Gemini, OpenAI API, Vertex AI, or any external LLM service.
2. Do not rely on LLM-as-a-judge for reward.
3. Do not implement MAPPA-style external LLM coach as a required component.
4. All rewards must be offline and reproducible.
5. Use only:
   - exact match
   - symbolic equivalence
   - unit tests
   - execution results
   - tool success/failure
   - retrieval hit / evidence overlap
   - format checks
   - local rule-based scorers
   - optional local model scorer only if already available on the server.
```

---

## 2. What to reuse from Dr. MAS

Do not rewrite Dr. MAS from scratch.

Reuse Dr. MAS components for:

```text
1. Agent registry
2. Multi-agent orchestration
3. Math and search environments
4. Agent-model assignment
5. LLM sharing / non-sharing mode
6. Per-agent configuration
7. Rollout collection
8. GRPO / Dr.MAS-style training loop
9. Existing agent-wise normalization implementation
10. Existing logging / trajectory format where possible
```

GraphCredit-Offline should be implemented as an extension layer.

Preferred integration style:

```text
Do not heavily rewrite existing Dr. MAS core.
Add a new package or module namespace:
    graphcredit_offline/

Then minimally patch Dr. MAS rollout and trainer calls to invoke GraphCredit hooks.
```

---

## 3. What GraphCredit-Offline adds

GraphCredit-Offline must add four core components:

```text
1. Event graph logger:
   Convert each multi-agent rollout into an EventGraph.

2. Offline process reward:
   Compute rule/verifier-based process scores for each trainable node.

3. Counterfactual credit:
   Estimate node-level marginal contribution by masking/replacing event nodes.

4. Agent-node-type advantage normalization:
   Normalize rewards by agent_id + node_type, not just agent_id.
```

## 5. Event graph schema

### 5.1 Node types

Support these node types:

```text
agent_message:
    A natural-language message generated by an agent.

agent_action:
    A normal action or decision generated by an agent.

tool_call:
    A tool invocation generated by an agent.

tool_result:
    Tool output. Not directly trainable. Its score is assigned to the parent tool_call.

router_decision:
    A planner/router/controller decision that chooses the next agent, tool, or branch.

verifier_judgment:
    A verifier/critic judgment about an intermediate result or final answer.

final_answer:
    Final answer generated by an answer agent or main agent.

memory_write:
    Optional. Agent writes to memory, scratchpad, or artifact store.

memory_read:
    Optional. Agent reads from memory, scratchpad, or artifact store.
```

### 5.2 Edge types

Support these edge types:

```text
temporal_edge:
    Event A happens before event B.

communication_edge:
    Message from one agent is consumed by another agent.

tool_edge:
    tool_call produces tool_result.

control_edge:
    router/planner decides to call an agent or branch.

evidence_edge:
    evidence/tool_result/message supports final_answer.

artifact_edge:
    file/memory/artifact produced by one event is used by another event.
```

## 6. Offline reward design

### 6.1 Node reward formula

For each trainable node:

```text
node_reward =
    alpha * global_reward
  + beta * process_reward(node)
  + gamma * counterfactual_credit(node)
  + delta * downstream_usage_score(node)
  - eta * cost_penalty(node)
  - zeta * redundancy_penalty(node)
```

Default weights:

```yaml
reward_weights:
  alpha_global: 0.30
  beta_process: 0.25
  gamma_counterfactual: 0.35
  delta_downstream_usage: 0.10
  eta_cost: 0.05
  zeta_redundancy: 0.05
```

Rationale:

```text
Counterfactual credit receives the highest weight because there is no external LLM coach.
Process reward is useful but should be rule-based and verifiable.
Global reward keeps all nodes aligned with final success.
```

## 8. Math task process scorers

Implement:

```text
graphcredit_offline/rewards/math_scorer.py
```

### 8.1 final_answer score

For final_answer nodes:

```text
score = 1.0 if final answer matches ground truth
score = 0.0 otherwise
```

Support:

```text
1. exact match
2. normalized numeric match
3. optional symbolic equivalence through local Python / sympy if installed
```

### 8.2 verifier_judgment score

For verifier_judgment nodes:

```text
If verifier says solution is correct and final answer is correct:
    score = 1.0

If verifier says solution is correct but final answer is wrong:
    score = 0.0

If verifier correctly rejects wrong intermediate solution:
    score = 1.0

If verifier rejects correct solution:
    score = 0.0
```

If intermediate correctness is unknown:

```text
fallback to final_reward-based weak signal:
    score = final_reward
```

### 8.3 solver message/action score

For solver action/message nodes:

Use offline heuristics:

```text
+0.3 if output contains a parseable final answer or intermediate equation
+0.2 if output follows required format
+0.2 if output is not empty and not repetitive
+0.2 if final_reward == 1
+0.1 if length is within reasonable range

subtract:
-0.2 if output is too long
-0.2 if output repeats previous message
-0.3 if output has malformed answer format
```

Clip to `[0, 1]`.

## 9. Search task process scorers

Implement:

```text
graphcredit_offline/rewards/search_scorer.py
```

Assume offline search/retrieval results are available in the dataset or local index.

### 9.1 tool_call score for search

For tool_call nodes:

```text
score components:
    entity_coverage_score
    query_specificity_score
    non_redundancy_score
    tool_success_score
    gold_evidence_hit_score
```

Suggested formula:

```text
score =
    0.25 * entity_coverage_score
  + 0.20 * query_specificity_score
  + 0.20 * non_redundancy_score
  + 0.15 * tool_success_score
  + 0.20 * gold_evidence_hit_score
```

Definitions:

```text
entity_coverage_score:
    whether query contains important entities from task_prompt.

query_specificity_score:
    penalize too short, too broad, or generic queries.

non_redundancy_score:
    penalize near-duplicate previous queries.

tool_success_score:
    1 if tool_result is non-empty and no error, else 0.

gold_evidence_hit_score:
    1 if retrieved result contains gold answer/evidence string, else 0.
```

### 9.2 answer node score

For final_answer nodes:

```text
score =
    0.70 * answer_correctness
  + 0.30 * evidence_support
```

Where:

```text
answer_correctness:
    exact match / normalized F1 / dataset evaluator.

evidence_support:
    final answer overlaps with retrieved evidence or gold evidence.
```

### 9.3 verifier judgment score

For verifier_judgment nodes:

```text
score = 1.0 if verifier correctly decides whether enough evidence exists
score = 0.0 otherwise
```

Approximation:

```text
If verifier stops search and final answer is correct:
    score = 1.0

If verifier stops search and final answer is wrong:
    score = 0.0

If verifier asks to continue search and current evidence lacks gold answer:
    score = 1.0

If verifier asks to continue search despite sufficient evidence:
    score = 0.3
```

## 11. Counterfactual credit

Implement:

```text
graphcredit_offline/rewards/counterfactual.py
```

### 11.1 Definition

```text
counterfactual_credit(node)
=
final_value(original_graph)
-
final_value(masked_or_replaced_graph)
```

Interpretation:

```text
credit > 0:
    node is useful.

credit = 0:
    node is redundant.

credit < 0:
    node is harmful.
```

### 11.2 MVP mode: offline masking only

Do not implement partial rerun or full rerun in MVP.

Offline masking:

```text
1. Copy EventGraph.
2. Replace target node output_content with a node-type-specific mask.
3. Remove evidence/control/communication edges from target node where appropriate.
4. Recompute final_value using an offline evaluator.
5. Return original_value - masked_value.
```

### 11.3 Replacement strategy

```text
agent_message:
    "[MESSAGE_REMOVED]"

agent_action:
    "[ACTION_REMOVED]"

tool_call:
    "[TOOL_CALL_REMOVED]"

tool_result:
    "[TOOL_RESULT_UNAVAILABLE]"

router_decision:
    "[ROUTER_DECISION_REMOVED]"

verifier_judgment:
    "[NEUTRAL_VERIFIER_JUDGMENT]"

final_answer:
    Do not counterfactually mask final_answer in MVP.
```

### 11.4 Which nodes to mask

Candidate selection:

```text
Do not mask every node.
Select top-k candidate nodes per trajectory.
```

Default:

```yaml
counterfactual:
  mode: offline_masking
  top_k: 5
```

Candidate score:

```text
candidate_score =
    0.35 * downstream_usage_score
  + 0.25 * node_type_priority
  + 0.20 * process_reward_uncertainty
  + 0.20 * cost_penalty
```

Node priority:

```text
tool_call: 1.0
router_decision: 1.0
verifier_judgment: 0.9
agent_action: 0.8
agent_message: 0.6
memory_write: 0.5
memory_read: 0.4
tool_result: 0.0, assign to parent tool_call
final_answer: 0.0, do not select
```

### 11.5 Counterfactual evaluator

Do not use LLM judge.

Use task-specific offline evaluator:

```text
math:
    recompute whether remaining graph still contains sufficient correct answer support.
    fallback: if masked node was on evidence path to final answer, reduce final value.

search:
    if masked node removes gold evidence or evidence used by final answer, final_value drops.
    if masked node is not used downstream, final_value remains.

code:
    rerun local unit tests if possible.
    otherwise use compile/test result stored in graph.

debate:
    if masked critique/refinement changes whether final answer is correct in offline rerun, use correctness.
    MVP may use static evaluator based on final correctness and dependency path.
```


Definition:

```text
downstream_usage_score =
    0.40 * direct_reference
  + 0.30 * dependency_path_score
  + 0.30 * final_answer_overlap
```

Offline implementations:

```text
direct_reference:
    1 if downstream node input_context contains this node_id or explicit consumed_node_ids.

dependency_path_score:
    normalized number of downstream paths from node to final_answer.

final_answer_overlap:
    lexical token overlap between node output_content and final_answer.
```

No embedding API required.

Optional local embedding model may be supported only if config enables it.

## 13. Cost and redundancy penalties

Implement:

```text
graphcredit_offline/rewards/cost.py
```

### 13.1 cost_penalty

```text
cost_penalty =
    0.4 * token_cost
  + 0.3 * tool_cost
  + 0.2 * repetition_cost
  + 0.1 * latency_cost
```

Definitions:

```text
token_cost:
    min(output_tokens / token_budget, 1)

tool_cost:
    1 if node_type == tool_call else 0

repetition_cost:
    lexical similarity with previous same-agent node

latency_cost:
    normalized local runtime if available, else 0
```

### 13.2 redundancy_penalty

```text
if counterfactual_credit is close to 0
and downstream_usage_score is low
and cost_penalty is high:
    redundancy_penalty = 1
else:
    redundancy_penalty = 0
```

Default thresholds:

```yaml
redundancy:
  credit_abs_threshold: 0.05
  usage_threshold: 0.20
  cost_threshold: 0.50
```

## 15. Advantage normalization

Implement:

```text
graphcredit_offline/rl/normalizer.py
```

### 15.1 Default bucket

```text
bucket = agent_id + node_type
```

Examples:

```text
Solver_agent_message
Verifier_verifier_judgment
Searcher_tool_call
Answerer_final_answer
Planner_router_decision
```

### 15.2 Formula

```text
advantage(node)
=
(node_reward - bucket_mean)
/
(bucket_std + epsilon)
```

### 15.3 Fallback

If bucket size is too small:

```text
1. fallback to agent_id
2. fallback to role + node_type
3. fallback to node_type
4. fallback to global
```

Default config:

```yaml
normalization:
  bucket: agent_id+node_type
  epsilon: 1.0e-6
  min_std: 1.0e-3
  min_bucket_size: 4
```

### 15.4 Dr. MAS integration

Dr. MAS already normalizes advantages by agent.

Modify it as follows:

```text
Original Dr. MAS:
    bucket = agent_id

GraphCredit-Offline:
    bucket = agent_id + node_type
```

Do not remove the original Dr. MAS agent-wise normalizer.

Instead:

```text
1. Add a config option:
   normalization.mode = "agent" | "agent_node_type"

2. Keep "agent" as baseline.

3. Add "agent_node_type" as GraphCredit default.
```

### 17.1 Main metrics

```text

    original task success.

    accuracy per rollout / token / tool call.

    variance of node_reward by agent and node_type.

    max gradient norm, std of gradient norm, number of spikes.

    fraction of nodes with counterfactual_credit < 0.

redundant_node_ratio:
    fraction of nodes with abs(counterfactual_credit) < threshold and high cost.

counterfactual_faithfulness:
    average value drop after masking top-credit nodes
    minus average value drop after masking random nodes.
```

```yaml
    type: "offline_rule"
    task_type: "math"

  counterfactual:
    enabled: true

  normalization:
    mode: "agent_node_type"

    min_std: 1.0e-3
    min_bucket_size: 4

  metrics:
    compute_faithfulness: true
    compute_harmful_nodes: true
    compute_redundant_nodes: true

  llm_judge:
    enabled: false
```

## 24. Coding rules for Codex

Codex must follow these rules:

```text
1. Do not add external API calls.
2. Do not use online LLM judge.
3. Do not remove original Dr. MAS training code.
4. Implement GraphCredit as an extension layer.
5. Add tests for every new module.
6. Every reward must produce a RewardBreakdown.
7. Every counterfactual result must be cached or saved.
8. Every random choice must accept a seed.
9. Every graph must be serializable to JSON.
10. Do not hard-code only the math topology.
11. Do not make tool_result a trainable node.
12. Do not replace Dr. MAS agent-wise normalization; add agent-node-type normalization as an option.
```

## 26. Paper contribution wording

Use the following contribution framing:

```text
1. Event-graph credit assignment:
   We extend multi-agent LLM RL by representing rollouts as event graphs and assigning credit to action/message/tool-call/router/verifier/final-answer nodes.

2. Offline verifiable process rewards:
   Instead of relying on external LLM coaches, we design task-specific rule/verifier-based process rewards for math and search agents.

3. Counterfactual node credit:
   We estimate the marginal contribution of event nodes using offline masking and task-specific final-value evaluators.

4. Agent-node-type advantage normalization:
   We extend Dr. MAS agent-wise normalization to agent-node-type normalization, reducing reward-scale mismatch across heterogeneous actions.

5. Empirical validation:
   We evaluate on Dr. MAS math and search environments, comparing against original Dr. MAS and reward/normalization ablations.
```

## 27. One-sentence project summary

```text
GraphCredit-Offline builds on Dr. MAS by adding event-graph logging, offline verifiable node rewards, counterfactual masking credit, and agent-node-type advantage normalization, enabling finer and more reproducible credit assignment for multi-agent LLM reinforcement learning without external LLM judges.
```
